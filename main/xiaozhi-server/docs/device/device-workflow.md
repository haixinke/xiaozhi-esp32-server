# 设备联调流程图解

## ⚠️ 重要提示

**WebSocket 连接地址应从配置接口动态获取**

在实际开发中，设备端应：
1. 调用 `POST /config/agent-models` 接口
2. 从响应的 `server.websocket` 字段获取实际连接地址
3. 使用获取的地址建立 WebSocket 连接

**原因**：
- ✅ 支持不同部署环境（Docker/公网/域名）
- ✅ 配置集中管理，无需固件升级
- ✅ 生产环境可使用域名和 SSL

本流程图中使用的 `ws://HOST:8000` 仅作为示例，实际使用时请从配置接口获取。

## 关键检查点

### ✅ 检查点 1: 环境准备

```
□ 服务器可访问 (ping 测试)
□ 端口正常开放 (8000, 8002)
□ MAC 地址已记录
□ 管理后台可登录
```

### ✅ 检查点 2: 设备注册

```
□ 调用注册 API 成功
□ 获得 6 位验证码
□ 验证码已显示或记录
```

### ✅ 检查点 3: 设备绑定

```
□ 在管理后台输入验证码
□ 选择正确的智能体
□ 绑定成功提示
□ 设备列表显示新设备
```

### ✅ 检查点 4: WebSocket 连接

```
□ TCP 连接成功
□ WebSocket 握手成功
□ Hello 消息发送成功
□ 收到 session_id
```

### ✅ 检查点 5: 音频交互

```
□ 音频编码参数正确 (Opus, 24kHz)
□ VAD 检测正常
□ ASR 识别返回文本
□ LLM 对话返回内容
□ TTS 合成返回音频
```

### ✅ 检查点 6: 保活机制

```
□ Ping 消息定时发送
□ Pong 响应正常
□ 连接不会频繁断开
```

## 测试用例

### 用例 1: 基础连接测试

```yaml
名称: "建立 WebSocket 连接"
前置条件:
  - 设备已注册并绑定
步骤:
  1. 连接到 ws://HOST:8000/xiaozhi/v1/?device-id=XXX
  2. 发送 Hello 消息
  3. 等待服务器响应
预期结果:
  - 连接成功建立
  - 收到 session_id
  - 服务器日志显示设备上线
```

### 用例 2: 简单对话测试

```yaml
名称: "发送一句话并收到回复"
前置条件:
  - WebSocket 已连接
  - Hello 握手完成
步骤:
  1. 录制 "你好小智"
  2. 编码为 Opus 格式
  3. 发送到服务器
  4. 等待响应
预期结果:
  - 收到 TTS 音频
  - 音频可以正常播放
  - 内容相关且合理
```

### 用例 3: 中断测试

```yaml
名称: "中断正在播放的音频"
前置条件:
  - 正在播放 TTS 音频
步骤:
  1. 发送 Abort 消息
  2. 观察音频播放
预期结果:
  - 音频立即停止
  - 可以开始新的对话
```

### 用例 4: 保活测试

```yaml
名称: "心跳保活机制"
前置条件:
  - WebSocket 已连接
步骤:
  1. 每 30 秒发送 Ping
  2. 记录 Pong 响应时间
  3. 持续 5 分钟
预期结果:
  - 每次 Ping 都收到 Pong
  - 连接不会断开
  - 响应时间 < 500ms
```
