## 概述

xiaozhi-esp32-server 使用基于**装饰器**和**模块自动导入**的插件系统，实现了工具函数的零配置自动注册。

### 核心优势

- ✅ **零配置**：添加新工具只需创建文件并装饰
- ✅ **自动发现**：启动时自动扫描并注册所有插件
- ✅ **类型安全**：通过 `ToolType` 枚举区分工具类型
- ✅ **统一管理**：所有函数集中在全局注册表
- ✅ **易于扩展**：新插件与现有代码完全解耦

---

## 核心概念

### 1. Python 装饰器（Decorator）

装饰器是 Python 的一种语法糖，用于在不修改函数代码的情况下增强函数功能。

#### 基本语法

```python
@decorator
def my_function():
    pass
```

#### 等价展开

```python
def my_function():
    pass
my_function = decorator(my_function)
```

#### 带参数的装饰器（装饰器工厂）

```python
@decorator(arg1, arg2)
def my_function():
    pass
```

**等价于：**

```python
temp = decorator(arg1, arg2)
def my_function():
    pass
my_function = temp(my_function)
```

### 2. 模块导入机制

当 Python 导入一个模块时，会执行该模块的所有顶层代码：

```python
import my_module  # 会执行 my_module.py 的所有代码
```

**执行的代码包括：**
- 所有 `import` 语句
- 所有函数定义（`def`）
- 所有类定义（`class`）
- 所有装饰器应用（`@decorator`）
- 所有顶层代码（赋值、函数调用等）

### 核心组件

#### 1. 注册表（plugins_func/register.py）

```python
# 全局函数注册字典
all_function_registry = {}

class FunctionItem:
    """函数项，封装函数及其元数据"""
    def __init__(self, name, description, func, type):
        self.name = name
        self.description = description
        self.func = func
        self.type = type

def register_function(name, desc, type=None):
    """装饰器工厂：创建装饰器"""
    def decorator(func):
        # 将函数注册到全局字典
        all_function_registry[name] = FunctionItem(name, desc, func, type)
        return func  # 返回原函数，不改变其行为
    return decorator
```

#### 2. 自动发现（plugins_func/loadplugins.py）

```python
import importlib
import pkgutil

def auto_import_modules(package_name):
    """自动导入指定包内的所有模块"""
    # 获取包路径
    package = importlib.import_module(package_name)
    package_path = package.__path__

    # 遍历并导入所有模块
    for _, module_name, _ in pkgutil.iter_modules(package_path):
        full_module_name = f"{package_name}.{module_name}"
        importlib.import_module(full_module_name)  # 触发装饰器执行
```

#### 3. 工具管理器（core/providers/tools/unified_tool_manager.py）

```python
class ToolManager:
    """统一工具管理器"""

    def __init__(self, conn):
        self.executors: Dict[ToolType, ToolExecutor] = {}
        self._cached_tools: Optional[Dict[str, ToolDefinition]] = None

    def register_executor(self, tool_type: ToolType, executor: ToolExecutor):
        """注册工具执行器"""
        self.executors[tool_type] = executor

    def get_all_tools(self) -> Dict[str, ToolDefinition]:
        """获取所有工具定义"""
        all_tools = {}
        for executor in self.executors.values():
            tools = executor.get_tools()
            all_tools.update(tools)
        return all_tools

    async def execute_tool(self, tool_name: str, arguments: Dict[str, Any]):
        """执行工具调用"""
        tool_type = self.get_tool_type(tool_name)
        executor = self.executors[tool_type]
        return await executor.execute(self.conn, tool_name, arguments)
```

#### 4. 执行器（core/providers/tools/server_plugins/plugin_executor.py）

```python
class ServerPluginExecutor(ToolExecutor):
    """服务端插件工具执行器"""

    def get_tools(self) -> Dict[str, ToolDefinition]:
        """获取所有注册的服务端插件工具"""
        tools = {}
        for func_name, func_item in all_function_registry.items():
            tools[func_name] = ToolDefinition(
                name=func_name,
                description=func_item.description,
                tool_type=ToolType.SERVER_PLUGIN
            )
        return tools

    async def execute(self, conn, tool_name: str, arguments: Dict[str, Any]):
        """执行工具函数"""
        func_item = all_function_registry.get(tool_name)
        result = func_item.func(**arguments)
        return result
```

## 实现细节

### ToolType 枚举

```python
class ToolType(Enum):
    """工具类型枚举"""
    SERVER_PLUGIN = "server_plugin"  # 服务端插件
    SERVER_MCP = "server_mcp"        # 服务端 MCP
    DEVICE_IOT = "device_iot"        # 设备端 IoT
    DEVICE_MCP = "device_mcp"        # 设备端 MCP
    MCP_ENDPOINT = "mcp_endpoint"    # MCP 接入点
```

### Action 枚举

```python
class Action(Enum):
    """工具执行后的动作类型"""
    ERROR = -1       # 错误
    NOTFOUND = 0     # 工具未找到
    NONE = 1         # 啥也不干
    RESPONSE = 2     # 直接回复用户
    REQLLM = 3       # 调用函数后再请求 LLM 生成回复
```

### ActionResponse 类

```python
class ActionResponse:
    """工具执行响应"""
    def __init__(self, action: Action, result=None, response=None):
        self.action = action      # 动作类型
        self.result = result      # 动作产生的结果
        self.response = response  # 直接回复的内容
```

## 完整工作流程

### 1. 应用启动阶段

```
启动应用
  ↓
UnifiedToolHandler.__init__()
  ↓
创建 ToolManager
  ↓
注册各类执行器（ServerPluginExecutor、ServerMCPExecutor 等）
  ↓
调用 auto_import_modules("plugins_func.functions")
  ↓
扫描 plugins_func/functions/ 目录
  ↓
  ├─ 发现 get_weather.py
  │   ↓
  │ importlib.import_module("plugins_func.functions.get_weather")
  │   ↓
  │ 【Python 开始执行 get_weather.py】
  │   ↓
  │ 1. 导入语句
  │ 2. 定义 WEATHER_DESC
  │ 3. 遇到 @register_function("get_weather", ...)
  │     ↓
  │   【装饰器执行】
  │     ↓
  │   register_function("get_weather", ...)  # 调用工厂
  │     ↓
  │   返回 decorator 函数
  │     ↓
  │   decorator(get_weather)  # 用函数调用装饰器
  │     ↓
  │   all_function_registry["get_weather"] = FunctionItem(...)
  │     ↓
  │   返回 get_weather 函数
  │     ↓
  │ 4. 函数定义完成
  │   ↓
  │ 【模块导入完成，函数已注册】
  │
  ├─ 发现 get_time.py
  │   ↓
  │ 【同样流程...】
  │
  └─ 发现 play_music.py
      ↓
    【同样流程...】
  ↓
所有插件函数已注册到 all_function_registry
  ↓
ToolManager 初始化完成
```

### 2. 运行时调用阶段

```
LLM 决定调用工具
  ↓
handle_llm_function_call("get_weather", {"location": "北京"})
  ↓
ToolManager.execute_tool("get_weather", {"location": "北京"})
  ↓
查找工具类型：get_weather → SERVER_PLUGIN
  ↓
获取执行器：ServerPluginExecutor
  ↓
ServerPluginExecutor.execute(conn, "get_weather", {"location": "北京"})
  ↓
从 all_function_registry 获取函数
  ↓
根据 ToolType 决定调用方式：
  - SYSTEM_CTL(4): func(conn, **arguments)
  - IOT_CTL(5): func(conn, **arguments)
  - WAIT(2): func(**arguments)
  - 其他: func(**arguments)
  ↓
执行 get_weather(location="北京")
  ↓
返回 ActionResponse(Action.REQLLM, "北京天气：晴天", None)
  ↓
UnifiedToolHandler 处理响应
  ↓
返回给 LLM 或直接回复用户
```

## 使用示例

### 示例 1：简单的工具函数

```python
# plugins_func/functions/get_time.py

from plugins_func.register import register_function, ToolType, ActionResponse, Action
from datetime import datetime

GET_TIME_DESC = {
    "type": "function",
    "function": {
        "name": "get_time",
        "description": "获取当前时间",
        "parameters": {
            "type": "object",
            "properties": {},
            "required": []
        }
    }
}

@register_function("get_time", GET_TIME_DESC, ToolType.WAIT)
def get_time():
    """获取当前时间"""
    now = datetime.now()
    time_str = now.strftime("%Y-%m-%d %H:%M:%S")
    return ActionResponse(Action.REQLLM, f"当前时间是：{time_str}", None)
```

### 示例 2：带参数的工具函数

```python
# plugins_func/functions/get_weather.py

from plugins_func.register import register_function, ToolType, ActionResponse, Action
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from core.connection import ConnectionHandler

GET_WEATHER_DESC = {
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "获取指定地点的天气信息",
        "parameters": {
            "type": "object",
            "properties": {
                "location": {
                    "type": "string",

    return ActionResponse(Action.REQLLM, weather_report, None)
```

## 扩展指南

### 添加新的工具函数

#### 步骤 1：创建新文件

在 `plugins_func/functions/` 目录下创建新文件，例如 `my_tool.py`：

```python
# plugins_func/functions/my_tool.py

from plugins_func.register import register_function, ToolType, ActionResponse, Action

# 1. 定义函数描述（OpenAI 函数调用格式）
MY_TOOL_DESC = {
    "type": "function",
    "function": {
        "name": "my_tool",
        "description": "工具功能的简要描述",
        "parameters": {
            "type": "object",
            "properties": {
                "param1": {
                    "type": "string",
                    "description": "参数1的说明"
                },
                "param2": {
                    "type": "integer",
                    "description": "参数2的说明"
                }
            },
            "required": ["param1"]
        }
    }
}

# 2. 使用装饰器注册函数
@register_function("my_tool", MY_TOOL_DESC, ToolType.WAIT)
def my_tool(param1: str, param2: int = 10):
    """
    工具函数的实现

    Args:
        param1: 参数1说明
        param2: 参数2说明，默认值10

    Returns:
        ActionResponse: 执行结果
    """
    # 实现业务逻辑
    result = f"执行结果：{param1} + {param2}"

    # 返回 ActionResponse
    return ActionResponse(Action.REQLLM, result, None)
```

#### 步骤 2：配置工具描述（可选）

在 `config.yaml` 或 `data/.config.yaml` 中添加自定义描述：

```yaml
plugins:
  my_tool:
    description: "自定义的工具描述，将覆盖代码中的默认描述"
```

#### 步骤 3：重启应用

重启应用后，新工具会自动被发现并注册。

#### 步骤 4：验证

查看启动日志，确认工具已注册：

```
当前支持的函数列表: ['get_weather', 'get_time', 'my_tool', ...]
```

### 选择合适的 ToolType

根据工具的功能选择合适的类型：

| ToolType | 代码 | 说明 | 是否需要 conn 参数 |
|----------|------|------|-------------------|
| NONE | 1 | 调用完工具后，不做其他操作 | 否 |
| WAIT | 2 | 调用工具，等待函数返回 | 否 |
| CHANGE_SYS_PROMPT | 3 | 修改系统提示词，切换角色 | 是 |
| SYSTEM_CTL | 4 | 系统控制，如退出、播放音乐等 | 是 |
| IOT_CTL | 5 | IoT 设备控制 | 是 |
| MCP_CLIENT | 6 | MCP 客户端 | 否 |

### 返回合适的 Action

根据工具执行结果选择合适的 Action：

| Action | 代码 | 说明 | 使用场景 |
|--------|------|------|----------|
| ERROR | -1 | 错误 | 工具执行失败 |
| NOTFOUND | 0 | 工具未找到 | 工具不存在 |
| NONE | 1 | 啥也不干 | 静默执行 |
| RESPONSE | 2 | 直接回复 | 不需要 LLM 生成回复 |
| REQLLM | 3 | 请求 LLM 生成回复 | 需要基于结果生成回复 |

## 常见问题

### Q1: 为什么我的工具没有被注册？

**检查清单：**

1. ✅ 文件是否在 `plugins_func/functions/` 目录下？
2. ✅ 是否使用了 `@register_function` 装饰器？
3. ✅ 装饰器的第一个参数（函数名）是否唯一？
4. ✅ 是否在 `config.yaml` 中启用了该工具？

### Q2: 如何调试工具函数？

**方法：**

```python
@register_function("my_tool", MY_TOOL_DESC, ToolType.WAIT)
def my_tool(param1: str):
    # 添加日志
    from config.logger import setup_logging
    logger = setup_logging()
    logger.info(f"my_tool 被调用，参数: {param1}")

    # 打印调试信息
    print(f"DEBUG: param1 = {param1}")

    # 实现逻辑
    result = f"处理结果：{param1}"

    return ActionResponse(Action.REQLLM, result, None)
```

### Q3: ToolType.SYSTEM_CTL 和 ToolType.WAIT 有什么区别？

**区别：**

- `ToolType.WAIT`：不需要 `conn` 参数，直接调用 `func(**arguments)`
- `ToolType.SYSTEM_CTL`：需要 `conn` 参数，调用 `func(conn, **arguments)`

**示例：**

```python
# WAIT 类型 - 不需要 conn
@register_function("get_time", ..., ToolType.WAIT)
def get_time():
    return ActionResponse(Action.REQLLM, "当前时间...", None)

# SYSTEM_CTL 类型 - 需要 conn
@register_function("get_weather", ..., ToolType.SYSTEM_CTL)
def get_weather(conn, location):
    # 可以访问 conn.config、conn.client_ip 等
    return ActionResponse(Action.REQLLM, "天气信息...", None)
```

### Q4: 如何让工具支持异步？

**当前实现是同步的，如果需要异步：**

```python
import asyncio

@register_function("my_async_tool", {...}, ToolType.WAIT)
def my_async_tool(param1: str):
    # 在同步函数中运行异步代码
    result = asyncio.run(async_operation(param1))
    return ActionResponse(Action.REQLLM, result, None)

async def async_operation(param1: str):
    # 异步操作
    await asyncio.sleep(1)
    return f"异步处理结果：{param1}"
```

### Q5: 如何在工具中访问配置？

**方法：**

```python
@register_function("my_tool", {...}, ToolType.SYSTEM_CTL)
def my_tool(conn, param1: str):
    # 访问配置
    config = conn.config

    # 获取插件特定配置
    plugin_config = config.get("plugins", {}).get("my_tool", {})

    # 使用配置
    api_key = plugin_config.get("api_key", "default_key")

    return ActionResponse(Action.REQLLM, "结果...", None)
```

## 总结

xiaozhi-esp32-server 的工具函数注册机制通过以下三个核心概念实现：

1. **Python 装饰器**：`@register_function` 语法糖
2. **模块导入机制**：`import` 会执行模块代码
3. **全局注册表**：`all_function_registry` 字典

**核心流程：**

```
启动 → 扫描目录 → 导入模块 → 触发装饰器 → 自动注册 → 完成
```

这种设计使得添加新工具变得极其简单：创建文件、定义函数、添加装饰器，三步即可完成！
