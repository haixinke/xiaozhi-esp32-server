# AI 宠物孵化演示设计文档

**日期：** 2026-05-08
**状态：** 已批准
**项目：** xiaozhi-esp32-server / main/demo-web/

## 概述

将演示项目（demo-web）从单一测试页面改造为 AI 宠物孵化过程的沉浸式演示。面向终端用户展示从蛋到宠物的完整孵化体验，最终过渡到 AI 语音聊天交互。

## 目标受众

终端用户（非开发者），重点在视觉效果和仪式感。

## 技术方案

单页面状态机 + CSS/SVG 动画，零外部依赖。

## 孵化流程（五阶段状态机）

### 阶段 1：蛋（静态展示）

- 屏幕中央显示精致蛋，柔和呼吸光晕动画
- 底部有"开始孵化"按钮

### 阶段 2：孵化（用户互动）

- 用户点击/触摸蛋加速孵化
- 每次点击增加 3~5% 进度（随机）
- 自动增长约 0.5%/秒（不点也能孵完）
- 蛋的摇晃幅度和裂纹随进度递增：
  - 0~30%：轻微摇晃，偶尔晃动
  - 30~60%：摇晃加剧，出现细小裂纹
  - 60~90%：剧烈摇晃，裂纹增多变粗
  - 90~100%：蛋壳发光，即将破壳
- 点击反馈：弹性缩放 + 爱心粒子向上飘散 + "+N%" 飘字

### 阶段 3：破壳（自动播放）

- 进度满后自动触发
- 蛋壳裂开，粒子飞散
- 破壳动画播放到 50% 时发起 API 请求

### 阶段 4：出生展示

- 调用 `POST /xiaozhi/pet/birth`，请求体 `{ "deviceId": "当前设备ID" }`
- API 返回后，兔子从蛋壳中跳出
- 展示宠物信息卡（昵称、MBTI、星座、出生时间）
- 展示 3~4 秒后自动淡出，进入聊天界面
- 错误处理：API 失败显示 Toast 提示，仍展示兔子但用默认数据；设备未注册提示"请先配置设备信息"

### 阶段 5：聊天交互

- 复用现有 test_page.html 的完整功能
- 兔子形象替代 Live2D 角色，出现在左下角作为固定陪伴
- 保留：WebSocket 连接（OTA 流程）、语音录制 + Opus 编解码、聊天弹幕、设置面板、背景切换、MCP 工具
- 底部有"重新孵化"按钮，点击完全重置到阶段 1（重新调用出生 API）

## 兔子在聊天中的行为

- 待机：呼吸动画 + 偶尔眨眼
- 说话中（TTS 播放）：嘴巴张合动画
- 倾听中（录音）：耳朵竖起动画
- 收到消息：开心跳动

## 视觉设计

### 风格

卡通可爱风，暖色调。

### 蛋

- 椭圆形，上窄下宽
- 暖奶油色 (#FFF5E6)，描边 (#F0D9B5)
- 左侧椭圆高光 + 小光点，增加立体感
- 两侧淡粉色腮红 (#FFB4A2)，增加可爱感
- 柔和投影 (drop-shadow)
- 呼吸动画：缓慢缩放 + 光晕明暗变化
- 裂纹：SVG path 逐渐绘制，从顶部向下蔓延

### 兔子

- CSS/SVG 绘制
- 白色圆身，长耳朵内粉色 (#FFB6C1)
- 大眼睛带高光点，粉色鼻子，微笑弧线嘴
- 两侧腮红，与蛋的腮红呼应
- 圆润小身体 + 圆尾巴
- 动画：呼吸、眨眼、耳朵微动、左右摇晃

### 配色

| 用途 | 色值 |
|------|------|
| 蛋身 | #FFF5E6 |
| 描边 | #F0D9B5 |
| 粉色 | #FFB6C1 |
| 白色 | #FFFFFF |
| 腮红 | #FFB4A2 |
| 背景 | #FFF0F5 |

## 约束

- 仅浏览器运行（PC + 手机 H5）
- 纯静态 HTML 项目，不引入外部动画库
- 破壳时调用真实后端 `/pet/birth` API
- 循环时完全重置（包括重新调用出生 API）
- 保留原始 test_page.html 不改动
